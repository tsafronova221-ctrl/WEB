# Инструкция по новым функциям анти-списывания

## 📋 Обзор изменений

В систему добавлены следующие функции для предотвращения списывания во время контрольных работ:

### 1. ⏱️ Таймер времени выполнения
- **Настройка**: В админ-панели при редактировании лабораторной работы в режиме "Контрольная работа" появляется поле "Время на тест (минут)"
- **Работа**: 
  - Таймер отображается в правом верхнем углу экрана студента
  - Время сохраняется в localStorage браузера - при обновлении страницы таймер НЕ сбрасывается
  - За 5 минут до конца таймер начинает мигать красным
  - По истечении времени работа автоматически отправляется

### 2. 🚫 Отслеживание нарушений

Система отслеживает три типа нарушений:

#### а) Переключение вкладок (Tab Switching)
- Фиксируется каждый раз, когда студент переключается на другую вкладку или окно
- События `visibilitychange` и `blur` отслеживаются
- Студент видит предупреждение о зафиксированном нарушении

#### б) Копирование текста (Copy Detection)
- Фиксируется попытка копирования текста с страницы теста
- Событие `copy` перехватывается
- Студент видит предупреждение "Обнаружено копирование!"

#### в) Выход из полноэкранного режима (Fullscreen Exit)
- При начале теста браузер пытается перейти в полноэкранный режим
- Каждый выход из него фиксируется

### 3. 👁️ Просмотр нарушений в админ-панели

При просмотре попытки студента (`/admin/labs/<lab_id>/groups/<group_id>` → клик на попытку):
- Вверху страницы отображается блок "⚠️ Нарушения во время тестирования"
- Показываются:
  - Количество переключений вкладок
  - Факт копирования текста
  - Количество выходов из полноэкранного режима
- Если нарушений нет - отображается "✅ Нарушений не зафиксировано"

## 🔧 Технические детали

### Изменения в базе данных

**Таблица `labs`:**
- `test_duration INTEGER` - время на тест в минутах (0 = не ограничено)

**Таблица `attempts`:**
- `violation_tab_switch INTEGER` - количество переключений вкладок
- `violation_copy BOOLEAN` - было ли копирование
- `violation_fullscreen_exit INTEGER` - выходы из fullscreen

### Измененные файлы

1. `app/models.py` - добавлены новые поля в модели Lab и Attempt
2. `app/templates/public/questions.html` - таймер и защита от списывания (JavaScript)
3. `app/public/routes.py` - сохранение данных о нарушениях
4. `app/admin/labs/edit.py` - обработка test_duration
5. `app/templates/admin/edit_lab.html` - UI для настройки времени теста
6. `app/admin/labs/attemts.py` - передача данных о нарушениях в шаблон
7. `app/templates/admin/one_attempt.html` - отображение нарушений

## 📝 Как использовать

### Для администратора/преподавателя:

1. Создайте или отредактируйте лабораторную работу
2. Включите режим "Контрольная работа"
3. Укажите желаемое время выполнения в минутах
4. Сохраните изменения

### Для студента:

1. Войдите в систему
2. Выберите группу и введите пароль варианта
3. Начните выполнение работы
4. Следите за таймером в правом верхнем углу
5. Не переключайтесь на другие вкладки!
6. Не копируйте текст!
7. Работайте в полноэкранном режиме

## ⚠️ Важные замечания

- Таймер использует localStorage браузера - очистка кэша может сбросить таймер
- Отслеживание работает только во время выполнения контрольной работы (is_test=True)
- Данные о нарушениях сохраняются только при завершении работы
- Полноэкранный режим может быть заблокирован некоторыми браузерами
